# 数据与隐私合规（维C4 · ⭐事实锚定）

> 本 reference 给合规官一份"医药数据三法框架 + 跨境出境"落地手册。法规标 🔍，不确定标 💭。当前维度最薄，已加厚。

## C4.1 三法框架与医药场景映射 🔍
| 法 | 定位 | 医药场景映射 |
|---|---|---|
| 《网络安全法》（2017-06-01 施行） | 网络运行安全、关键信息基础设施 | 药企内网/系统安全、日志与访问控制、等保合规 |
| 《数据安全法》（2021-09-01 施行） | 数据分类分级、重要数据保护 | 临研数据/真实世界数据作为"数据资源"的分类分级与流转管控 |
| 《个人信息保护法》PIPL（2021-11-01 施行） | 个人信息（含敏感）处理规则、跨境、主体权利 | 患者健康信息、受试者数据、营销画像的知情同意与合规处理 |

> 🔍 三法共同构成我国数据治理基础；医药行业同时落在卫健/药监行业监管下（如《人类遗传资源管理条例》）。

## C4.2 医药特殊数据类型与分类分级 🔍
- **患者健康数据**：病历、诊断、处方、随访——属**敏感个人信息**（一旦泄露易致人格尊严受损/人身安全）🔍（PIPL 第28条）。
- **RWE/真实世界数据**：真实世界研究来源数据，须来源合法、去标识化与用途限定 🔍。
- **临研数据**：受试者个人数据，须经伦理与知情同意，遵循 GCP 与《人类遗传资源管理条例》（涉人遗资源须审批/备案）🔍。
- **基因/人类遗传资源数据**：属重要/敏感数据，出境与对外提供受严格管控（人遗条例、科技部/卫健委）🔍。
- **分类分级**：一般个人信息 / **敏感个人信息**（PIPL 第28-32条，须单独同意、影响评估）🔍；重要数据依《数据安全法》另行认定 💭（行业重要数据目录以主管部门发布为准）。

## C4.3 数据出境三路径 🔍
依《个人信息保护法》第38条及配套规定，个人信息出境须具备下列条件之一：
1. **安全评估**：通过国家网信部门（CAC）组织的**数据出境安全评估** 🔍——适用于大量个人信息/重要数据/关键信息基础设施运营者。
2. **保护认证**：经专业机构进行的**个人信息保护认证** 🔍。
3. **标准合同备案**：与境外接收方订立**标准合同**并向网信部门**备案** 🔍（依据《个人信息出境标准合同办法》）。

> 🔍 重要数据出境原则上须走安全评估；人遗资源对外提供另有专门审批路径 💭。

## C4.4 合规审计清单 + 处理者义务 🔍
**处理者义务（PIPL）**：
- 告知—同意（敏感信息须**单独同意**）、最小必要、目的限定 🔍；
- 个人信息保护影响评估（PIA）并存档至少 3 年 🔍；
- 委托/共享/转让/出境须约束对方并留痕 🔍；
- 安全事件应急与**个保合规审计** 🔍。

**合规审计清单（可填）**：见 §C4.6。

## C4.5 跨境与 license-out 注意事项
- **license-out 涉国外买方**：受试者/真实世界数据出境须走 §C4.3 三路径之一；人遗资源对外提供须按《人类遗传资源管理条例》审批/备案 🔍。
- 合同中明确：数据范围、用途、境外接收方保护能力、再转移限制、rights of data subjects、违约与中止 🔍。
- 与 drug-bd 接口：数据合规作为 DD 必查项，IP 与数据权属、地域分离审视 🔍。
- 与处方量脱钩（PAP/随访数据不反哺处方激励）；接 drug-retail R6、drug-sales S7 🔍。
